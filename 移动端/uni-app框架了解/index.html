与 vue 的区别

不需要引入Vue-router

由框架解决。页面创建时，自动将页面路由加入 pages.json 文件中。页面的跳转使用

生命周期改动

相对于 vue 的实例生命周期，uni-app 对程序应用层面和页面层面增加对应的生命周期

箭头函数

注意不要在回调上或框架选项属性如 created上使用箭头函数，因为 this 绑定的不是 vue ，是未定义的。

跨平台框架限制

H5的vue语法得到全部支持，App 和小程序有功能限制。

data 属性

使用函数返回 return 对象

全局变量

有四种方式提供实现 vue 的全局变量

  • 1 公用模块：缺点是每次都需要引入
  • 2 挂载 Vue.prototype：扩展Vue 的原型
    • 建议统一加前缀。这样可以与当前页面区分开
  • 3 gloablData 这是小程序的实现方式
    • 在 App 上声明全局变量，但是在 Vue 中是没有的。可以在 App.vue 的 globalData属性上读写这个值
  • 4 Vuex 状态管理模式，集中式存储所有组件状态。需要自己学习

class 和 style 动态绑定语法

https://cn.vuejs.org/v2/guide/class-and-style.html

v-for 索引和 key

h5 模式从1 开始，其他平台从0开始。

key 最好是唯一标识，如果不写可以选择忽略警告。

事件处理器

支持全 Vue 事件， 绑定事件用 @

表单组件替换

非H5端不支持列表

各平台的条件编译

具体取值看文档

  • 5+APP 平台
  • 除了H5 和 微信小程序平台
  • static 静态目录的处理

H5 的事件定义扩展

uniapp 没有 document，使用 plus.globalEvent.addEventListener

在 manifest.json 中需要开启新编译器，即自定义组件模式”usingComponents”:true

HTML元素和组件

一些标签的变更

page 相当于 body， view相当于 div

特定组件（高度不可修改）

  • NavigationBar，导航栏， 44px高度（H5）
  • TabBar，底部选项卡， 50px

容器标签 Template 和 block

前者用于条件渲染，后者用于列表渲染

H5 平台不支持小程序组件

若要使用各个小程序平台的自定义组件，就在项目根目录下分别创建对应目录

Vue 组件使用

三步骤：导入，注册和使用

全局组件的使用，在 main.js 中注册

页面样式与布局

因为是跨端，所以推荐完全使用 flex 布局系统

页面尺寸单位使用 rpx

基准是 750 px，跟小程序的实现一致。

也可以使用 px，这样就不会随着设备变化

样式的处理

样式导入 @import 需要使用相对路径，用 ; 表示语句结束

避免 style 属性动态渲染，影响性能，如

三个CSS全局变量

图片和icon处理

类似 url(file)-loader 的处理

  • 图片小于 40kb， 转为 base64
  • 大于等于 40kb， 要手动转为 base64 ，挪到服务器上用URL引入
  • 本地图片仅支持 ~@语法，绝对路径
  • 网络路径必须加 https（特别是阿里巴巴矢量图库，默认是没有协议头）

依赖管理

也是支持 Npm 管理的

业务处理

获取上页面回传参数

设置全局数据和方法——内置了Vuex

onError 错误捕获回调

因为这不是严格的生命周期，但是可以在根组件下添加 onError 的回调函数

组件属性设置不生效

记录组件内部变化的值，在设置新值之前先设置为记录的当前值
